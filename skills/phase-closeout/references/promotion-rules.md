# Promotion Rules

Use this reference when deciding whether a retrospective lesson stays an observation or
becomes a standing workflow rule.

## Default Rule

- First occurrence: record as **Observation**
- Second occurrence of the same pattern: **Promote**

## Promotion Workflow

1. Read the current `SpecRetro.Lessons_Learned.md`.
2. Search for the same pattern or recommendation theme.
3. If it already exists:
   - update the earlier entry status to `Promoted -> [destination]`
   - cite the second sprint occurrence
   - write the promoted rule into the destination doc
4. If it does not exist:
   - append a new `SRL-NNN`
   - set status to `Observation`

## Good Promotion Targets

- workflow reference
- dispatch template
- agent reference / routing docs
- lessons learned for workflow patterns

Do not promote one-off anecdotes.
