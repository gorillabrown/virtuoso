# Virtuoso v1.2.0 — Integration Design Spec

**Date:** 2026-06-29
**Status:** DRAFT — awaiting user review (brainstorming → spec gate)
**Goal:** Turn Virtuoso from a well-built *set of skills* into a *closed system* — every skill names its neighbors, the lifecycle loop closes, load-bearing rules live in one place, and the package gains its two missing thirds (an idea front door and a coherence-maintenance loop).
**Source:** ULTRATHINK + adversarial review (2026-06-29), grounded by four parallel evidence agents + direct source verification. Findings referenced by ID (BC-n blocking concern, G-n gap, F-n anti-pattern, Seam-n lifecycle seam).

---

## 1. Context & Motivation

Virtuoso's dispatch *core* (`roadmap-review` ↔ `next-pointer` ↔ `pointer-closeout` ↔ `roadmap-status`) is densely and correctly integrated. But the review found the package is not yet a closed system:

- **The executor floats free** — `virtuoso` never names `next-pointer` (its input) or `pointer-closeout` (its output) [BC-4/Seam-6].
- **The planning loop never closes** — `pointer-closeout` drains the eager-spec buffer one sprint at a time but never mentions it; nothing connects drain to refill [BC-5/Seam-2].
- **Islands** — `3rd-party-audit` (remediation bypasses the rubric) and `governance-sweep` (blind to the roadmap/queue; output orphaned) [Seam-4, Seam-1].
- **Orphaned modifiers** — `effort-levels` and `adversarial-review` are absent from the spec-*authoring* path where they're designed to be used [BC-6/F19/F20/F21].
- **Conflicting/duplicated load-bearing rules** — git ownership stated 3 incompatible ways [BC-2]; `git-handoff` (LEGACY) mandated by `pointer-closeout` [BC-1]; "CLI" means three different roles [BC-3]; the preflight (×6), dispatch rubric (×2), and KPI/LOE map (×3) are copied, not referenced [F5/F6/F8].
- **Two missing thirds** — no idea/backlog front door (the only entry point, `roadmap-update`, is an *external* plugin), and no owner for re-examining categorization over time.

## 2. Design Principles

1. **Correct coupling, not maximal coupling.** Close real loops; connect each island to its *one* natural neighbor; leave true utilities (`delayed-start`) alone.
2. **Named transitions.** Every lifecycle edge names its endpoint skill, so a reader can trace a sprint end-to-end.
3. **One source per rule (shared references via vendoring).** The genuine constraint is that skills load independently — satisfied by vendoring a reference file into `Virtuoso/` (the mechanism preflight already uses for `recalc.py`), *not* by copying text. "Self-contained" = "references a vendored file," not "carries its own copy."
4. **Detect vs. act.** Hygiene *detection* (read-only) and categorization *action* (judgment) are separate responsibilities owned by different skills — never the same logic in two places.
5. **Additive & reversible.** v1.2.0 is extensions and reconciliations, not rewrites.

## 3. Target Architecture

```
        ┌──────────────── Ideas (NEW) ───────────────┐
        │  capture full-detail → Virtuoso/Ideas/       │
        ▼                                              │ incorporate (status→Incorporated [SPRINT])
  roadmap-review ──names──▶ next-pointer ──names──▶ virtuoso ──names──▶ pointer-closeout
        ▲                       │ stub? escalate                         │
        │                       └───────────┐                blocker?    │ buffer<5? → recommend
        │                                   │            mid-dispatch ◀───┤   /roadmap-review
        │  lessons (named file) ◀───────────┼───────────────────────────-┘
        └──◀ roadmap-status (continual drift sense) ──escalate re-group──▶ roadmap-review
                         ▲
   governance-sweep (DETECT roadmap/queue hygiene) ──routes judgment items──┘
   3rd-party-audit ──remediation routed through──▶ roadmap-review (enters as rubric-passed sprints)

   Shared references (vendored into Virtuoso/references/): preflight · dispatch-rubric · KPI+LOE map · git-ownership · hygiene-checklist
   Modifiers wired at authoring seam: roadmap-review→effort-levels · next-pointer→adversarial-review
```

Three structural moves: **(A)** close the loop + bookend the executor; **(B)** collapse duplicated rules to vendored references; **(C)** wire modifiers at the authoring seam.

## 6. The Ideas Skill (detailed, locked)

- **Invocation:** `/idea`; triggers "capture an idea", "I have an idea", "park this", "add to ideas".
- **Flow (lite mirror of brainstorming):** light roadmap-context read → **guided-lite 3–5 questions** (problem → the idea → value → rough size/candidate Phase → dependencies) → write full-detail record → quick self-review (placeholder/ambiguity scan). No approaches/design-sections/approval-gate/writing-plans handoff.
- **Storage:** `Virtuoso/Ideas/IDEA-NNN-slug.md` + `Virtuoso/Ideas/INDEX.md` (one line per idea for retrieval). Mirrors `Close-Outs/`.
- **Record schema:** ID · title · date · source → problem/opportunity · **the idea (full detail)** · value · rough size (t-shirt) + candidate Phase/Stage · dependencies/related ideas · open questions · **status: Captured → Incorporated [SPRINT-CODE] / Parked / Dropped**.
- **Integration:** inbound user-invoked; outbound `roadmap-review` reads Captured ideas as its initiative source and flips adopted ones to `Incorporated [SPRINT-CODE]`; `governance-sweep` flags stale Captured ideas.

## 6A. Issue-Handoff Contract (virtuoso → mid-dispatch-decision) — v1.1.2

Today `virtuoso` Phase 5 says "STOP, reprint the plan, wait for direction" but names no successor and produces no artifact (Seam-5). This contract replaces that with a typed handoff.

**Producer — `virtuoso`.** Whenever execution hits an issue that must route to `mid-dispatch-decision` — a **stop, hold, block, elevation, or any decision the executor cannot make alone** — `virtuoso` MUST render the issue in this exact **7-field format**, both inline and as a saved file:

1. **tl;dr:** one line.
2. **Executive Summary:** 2–4 sentences — what happened and why it blocks.
3. **Evidence of issue:** concrete signals (errors, failing test, contradicted assumption, file:line, logs).
4. **Possible cause(s):** ranked hypotheses.
5. **Likely solution(s):** candidate fixes/paths.
6. **Confidence in cause and solution identification (1–10):** integer + one-line justification.
7. **Exported issue documentation path:** path to the saved `.md`.

**Artifact.** `virtuoso` writes an identical `.md` (fields 1–6; field 7 self-references the file) to `Virtuoso/Issues/Issue.<SPRINT-ID>.<YYYY-MM-DD>.md` (append `-N` if multiple on a date). That path is what field 7 reports. `Virtuoso/Issues/` is created by preflight.

**Consumer — `mid-dispatch-decision`.** Its primary input is now **a path to a virtuoso-produced issue `.md`**. Intake reads the 7 fields, runs the decision protocol against them, and writes the decision back — a `## Decision` block appended to the same issue file, plus the dispatch/staging amendments it makes today. The legacy "user pastes raw CLI output" path remains a fallback; the structured file is canonical.

**Why correctness, not polish:** it makes the executor→decision handoff a typed contract (a real artifact at a known path) instead of an unstructured paste, closes Seam-5 with evidence, and gives every escalation a durable, retrievable record.

## 8. Risks & Mitigations

- **Vendored references not present if preflight hasn't run.** Mitigation: skills already assume preflight (true for recalc.py); add a fallback note. Genuine constraint, accepted.
- **governance-sweep needs workbook-read it lacks.** Mitigation: reuse `openpyxl` (already a dependency); read-only.
- **Detection logic forking across status/sweep/review.** Mitigation: single `roadmap-queue-hygiene-checklist.md` referenced by all three (the whole point of D5).
- **Scope creep into a rewrite.** Mitigation: sprint gates; additive-only; each sprint independently shippable.

## 9. Validation

- **Loop-trace test:** a reader given only the skills can trace idea→capture→review→dispatch→execute→close-out→replenish, each transition naming its neighbor.
- **Zero-duplication test:** re-run the adjacency/duplication scan (Agent A method); isolated-node count drops; duplicate-block count = 0.
- **Per-sprint:** `validate.py` + `pytest` green; `bump_version --check` in sync.
- **Dark-horse:** verify the Catalog `Priority` absolute-range formula under headless (openpyxl) row insertion — out of scope for v1.2.0 but flagged.
